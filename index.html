1.2. Command

The command design pattern goal is to provide means for encapsulating requests as objects and decoupling the client that performs the request from the receiver that processes the request. In order to achieve such goal, the client creates one or more commands that are sent to an invoker and can be combined from simple single command operations to complex multiple command operations that the invoker will execute and optionally keep track of such execution.

1.2.1. Components

A possible implementation is composed by a client, a receiver, an invoker and one or more commands.

  • Client: responsible for the request creation. It creates and parametrizes one or more command instances and sends them to the invoker instance. 
      class Client
  {
      static void Main(string[] args)
      {
          var receiver = new Receiver();
          var command = new Command(receiver);
          var invoker = new Invoker(command);

          invoker.Execute();
      }
  }
  • Receiver: responsible for the request processing. It receives parameters encapsulated within a command instance and processes these parameters by executing its own logic. 
      class Receiver : IReceiver
  {
      public void Execute()
      {
          // Receiver logic
      }
  }
  • Invoker: responsible for executing one or more commands. It receives one or more command instances, executes them and optionally keeps track of the execution if undo and redo functionallity is required. 
      class Invoker
  {
      private ICommand _command;

      public Invoker(ICommand command)
      {
          _command = command;
      }

      public void Execute()
      {
          // Invoker logic

          _command.Execute();

          // Invoker logic
      }
  }
  • Command: responsible for receiving the receiver instance, the data required by the receiver and for calling the receiver. It encapsulates the request passed by a client through input parameters as though the receiver that will make use of these input parameteres, implements the actual logic for one or more actions that the abstract command stabilished as its contract for dealing with the receiver, invokes the receiver and returns the output parameters to the client that issued the request. 
      class Command : ICommand
  {
      private IReceiver _receiver;

      public Command(IReceiver receiver)
      {
          _receiver = receiver;
      }

      public void Execute()
      {
          _receiver.Execute();
      }
  }
