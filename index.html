1.2. Command

The command design pattern goal is to provide means for encapsulating requests as objects and to decouple the client that performs the request from the receiver that processes the request.

1.2.1. Class Diagram

A possible implementation is composed by a few components, namely...

  • Client: responsible for creating a request.
  • Receiver:
  • Invoker: responsible for processing a request.
  • Abstract Command: responsible for stabilishing the contract used by the client to create a request for the receiver.
  • Concrete Command: responsible for implementing the logic of the command.
